设计原则

亲密性

  • 如果某些元素在理解上存在关联，或者相互之间存在某种关系，那么这些元素在视觉上也应当有关联，除此之外，其他孤立的元素或元素组则不应存在亲密性。

  • 如果页面上有些区域的组织不太清晰，则要看看这些项是不是存在不该有的亲密性。

  • 根本目的是实现组织性。

  • 如果页面上的项超过3~5个（当然，这取决于具体情况），就要看看哪些孤立的元素可以归在一组建立更近的亲密性，使之成为一个视觉单元。

  • 要避免的问题

    1.避免一个页面上有太多的孤立元素

    2.不要再元素之间留出同样大小的空白，除非各组同属于一个子集

    3.不同属一组的元素之间不要建立关系

    4.不要仅仅因为有空白就把元素放在角落或中央

对齐

  • 每个元素应当与页面上的另外一个元素存在某种视觉联系。

  • 在设计中，统一性是一个重要的概念。

  • 根本目的

    • 使页面统一且有条理。

  • 如何实现

    • 在页面上找出与之对齐的元素，尽管这两个对象的物理位置可能相距很远。

  • 要避免的问题

    • 避免在页面上混合使用多种文本对齐方式。

    • 要着力避免居中对齐，除非你有意识地想要创建一种比较正式、稳重的表示。

重复

  • 根本目的

    • 统一，并增强视觉效果。

  • 如何实现

    • 可以找出现有的重复，然后将其加强。

  • 避免问题

    • 避免使用过多的重复。

对比

  • 页面上的不同元素之间要有对比效果，达到吸引读者的对比效果。

  • 要实现有效的对比，这两个元素必须截然不同 。

  • 根本目的

    • 增强页面的效果。

    • 有助于信息的组织 。

  • 如何实现

    • 字体选择、线宽、颜色、形状、大小、空间来增加对比。

  • 避免问题

    • 避免使用两种或多种类似的字体

颜色运用

  • 互补色

    • 色轮上完全对立的颜色，最佳搭配是一种作为主色，另一种用于强调。

  • 三色组

    • 彼此等距的三种颜色通常会形成一个让人愉悦的三色组

    • 所有三色组中的三种颜色都有基础色使其相互连接，因此看上去非常协调。

  • 分裂互补三色组

    • 从色轮的一边选择一种颜色，再在色轮上找出它对面的互补色，不过并不直接使用这个互补色，而是使用该互补色两侧的颜色。

  • 类似色

    • 由色轮上彼此相邻的颜色组成

  • 暗色和亮色。

    • 纯色就是色调

    • 向色调增加黑色就构成一个暗色

    • 向色调增加白色就构成一个亮色

  • 单色

    • 由一种色调及其相应的多种亮色和暗色组成。

  • 暗色和亮色的组合

  • 注意色质

    • 指某种颜色的特定明暗度、深浅度或色调。

    • 如果由于颜色组合而导致发生视觉抖动，这肯定只有一个原因，那就是色质上过于接近。

    • 如果色质很接近，看上去会有些模糊不清，对比太过微弱。

  • 暖色与冷色

    • 可以通过增加一些红色或黄色将某些颜色加热，也可以对颜色增加不同程度的蓝色对其“降温”。

    • 冷色趋于做背景色，而暖色是趋近型的。

    • 如果要组合暖色和冷色，一定要少用些暖色。

  • 取舍

  • CMYK与RGB

    • 组成RGB的有色光束并不是从任何物理物体反射来的，这是直接从显示器传入你眼睛的光。如果将所有颜色混合在一起会得到白色，如果去掉所有颜色，则得到黑色。

  • 印刷与WEB

字体设计

字体的基本原则

  • 引号

    • 开引号行似双6，闭引号形似双9。

    • 在美式英语中，逗号和句号总是出现在引号内的，英式英语，可内可外。

  • 撇号

  • 连接号

    • 连字符

    • 一字线

  • 破折号

  • 特殊符号

  • 重音符号

  • 大写字母

    • 为了强调而把词语变成全部大写并不一定是最好的办法，因为全部大写其实比小写字母更难阅读。

  • 下划线

    • 不要使用下划线。

  • 字距调整

    • 字距调整并不是要让字母都紧紧地贴在一起，而是要让字母间的空格看起来很连贯。

  • 寡妇和孤儿

    • 当一段文字的最后一行的字符数少于7个的时候（字符数取决于一行的长度），那么最后一行就成为寡妇。

    • 比在最后一行和下一个单词还糟糕的就是留下完整单词的一部分，而被连字号连接的前半部分还留在上一行。

  • 其他

    • 跟随在有样式的文字有的标点：如果有一个单词的样式是粗体、斜体或另外一种字体，紧随最后一个字母的标点也应该是同样的样式。

    • 括号中的标点

    • 段落缩进

    • 段落缩进或段落间的空行

    • 段首

    • 把文字放进方框里

    • 在列表中使用项目符号或装饰符号，而不是连字符

字体

  • 如果只使用一个字体系列，在样式、大小、字体粗线等方面没有什么变化，就会产生一种协调的关系。

  • 如果结合使用了多种字体，它们在样式、大小、字体粗细等方面很类似，但并不相同，就会出现一种冲突的关系。

  • 如果结合的多种字体和元素彼此之间截然不同，就会出现一种对比关系。

字体类别

  • Old style

    • 对角强调线
    • 比划中的粗细过渡缓和
    • 小写字母上的衬线是倾斜的

  • Modern

    • 没有括弧
    • 垂直强调线
    • 小写字母上的衬线很细，而且是水平的
    • 笔划中剧烈的粗细过渡

  • Slab serif（粗衬线体）

    • 笔划中粗细过渡或对比很小或没有
    • 小写字母上的衬线粗，且是水平的
    • 垂直强调线

  • Sans serif（无衬线体）

    • 笔划中没有粗细过渡
    • 没有衬线
    • 没有强调线，因为根本没有粗细变化

  • Script（手写体）

  • 花体

字体对比

  • 大小

  • 粗细

    • 指笔划的粗细度：常规、粗体，还包括：半粗体、极粗或者细字体。

    • 如果页面看上去“灰呼呼的”（有大篇幅的文字），而且没有空间来增加图片，也无法抽出引用文字把它们设置未图片，就可以使用非常粗的粗体设置关键短句。

    • 如果衬线体正文中间使用加粗的无衬线体强调关键语句，可能必须让加粗的无衬线体小一号，使之看上去与衬线体正文的大小相同。

  • 结构

    • 不要把同一个类别中的两种字体放在同一个页面上。

    • 必须选取两个不同字体类别中的字体。

    • 一种容易的方法就是选择一个衬线体和一个物衬线体，同时利用大小或粗细对比来强调差异。

  • 形状

    • 字符可能有相同的结构，但是“形状”不同。

    • 尽管全大写会降低可读性，但是如果你很肯定全大写对设计外观带来的好处更有意义，相比之下利大于弊，那么完全可以继续使用全大写。

    • 不要结合使用两种不同的斜体或者两种不同的手写体，也不要结合使用斜体和手写体。

  • 方向

    • 不要设置方向。

    • 千万不要在角落里放有角度的字体。

    • 有时字体方向的突然改变会产生显著的影响，或者提供一种独特的格式，这是使用字体方向的一种很好的理由。

    • 如果布局中可能存在方向的对比，那就做充分强调。

    • 明确的对齐线将突出并加强方向的对比。
